# 时序守望：基于微感知与时序分析的独居关怀系统

**项目愿景**：在保护隐私的前提下，通过非侵入式传感器数据与 AI Agent 技术，为独居老人提供从"异常检测"到"自动救援"的全流程闭环守护。

## 1. 项目简介

本系统是一套纯软件架构的智慧养老解决方案。系统通过模拟或接入真实的水、电、门磁等"微感知"数据，利用时序异常检测算法分析老人生活状态。其核心亮点在于引入了具备 **Function Calling** 能力的 **AI Agent** 作为中枢调度员，实现了从"算法预警"到"语音外呼"再到"社区自动派单"的自动化救援闭环。

### 核心特性

- **无感监测**：不使用摄像头，通过水电表数据波动分析行为，保护老人隐私。
- **双重检测**：结合"规则引擎（红线）"与"机器学习（基线偏离）"双重保障。
- **AI 闭环**：AI Agent 自动执行语音通知、超时自动向社区网格员派单。
- **亲情交互**：提供自然语言生成的"亲情日报"与 RAG 健康问答。

## 2. 系统架构与模块划分

系统采用微服务架构，前后端分离，主要包含以下 5 个核心模块：

| 模块编号 | 模块名称 | 技术栈 | 核心职责 |
| ------ |------ |------ |------ |
| **模块一** | **数据回放模拟器** | Python (Pandas) | 模拟 IoT 传感器数据，支持变速回放，用于测试与演示。 |
| **模块二** | **算法检测中枢** | Python (FastAPI, Sklearn) | 接收数据，运行 Isolation Forest 算法，输出红/黄预警信号。 |
| **模块三** | **Agent 调度中枢** | Python (FastAPI, SQL) | 接收预警，执行语音外呼、工单派发、状态流转的核心业务逻辑。 |
| **模块四** | **AI 交互服务** | Python (Flask) | 生成亲情日报，提供基于知识库的 AI 问答接口。 |
| **模块五** | **网格监控大屏** | Vue 3, ECharts | 社区端可视化界面，展示老人状态矩阵、报警时间轴。 |

## 3. 快速启动与部署指南

请按照以下顺序启动各模块，确保系统数据流转正常。

### 第一步：启动数据源 (模块一)

该模块用于模拟老人生活数据（水电门磁），向算法模块推送数据流。

- **路径**：`Module_1_Data_Simulator/`
- **依赖**：`pip install pandas requests`
- **配置**：修改 `FILE_INT` / `FILE_FLOAT` 指向本地 CSV 数据文件。
- **启动**：运行主程序，数据将自动推送到算法模块接口。

### 第二步：启动算法服务 (模块二)

负责处理时序数据并判断是否异常。

- **路径**：`Module_2_Algorithm/`
- **依赖**：`pip install fastapi uvicorn httpx pandas numpy scikit-learn pydantic`
- **启动命令**：

```
uvicorn main:app --host 0.0.0.0 --port 8001
```

- **注意**：确保配置中的 `DOWNSTREAM_URL` 指向模块三（Agent 中枢）的地址。

### 第三步：启动 Agent 调度中枢 (模块三)

系统的"大脑"，处理报警逻辑与数据库交互。

- **路径**：`Module_3_Agent_Core/`
- **依赖**：`pip install fastapi uvicorn sqlalchemy pymysql apscheduler`
- **数据库**：需配置 MySQL 连接字符串 (`database.py`)，首次启动会自动建表。
- **启动命令**：

```
python main.py
```

- **功能**：监听 8000 端口，处理外呼逻辑与工单状态流转。

### 第四步：启动 AI 交互服务 (模块四)

提供日报生成与问答能力。

- **路径**：`Module_4_AI_Service/`
- **依赖**：`pip install flask flask-cors requests`
- **启动命令**：`python app.py`
- **配置**：需配置阿里云 API Key 参及本地知识库文件。

### 第五步：启动前端监控大屏 (模块五)

用户可视化界面。

- **路径**：`Module_5_Frontend/`
- **运行方式**：直接浏览器打开 `grid_monitor.html`。
- **配置**：修改文件内的 `mainApiUrl` (指向模块三) 和 `dailyReportApiUrl` (指向模块四)。

## 4. 业务流转逻辑

1. **数据接入**：模块一模拟传感器数据 -> HTTP POST -> 模块二。
2. **异常检测**：
    - **红色预警**（如连续 12 小时无活动）：立即触发。
    - **黄色预警**（如作息偏离基线）：机器学习模型判定。
3. **Agent 调度**：
    - 模块二检测到异常 -> POST -> 模块三 (`/api/alert/from-detection`)。
    - 模块三入库并启动流程：
        - **RED**：模拟外呼家属 -> 等待确认 -> (若超时) 自动生成工单派发给网格员。
        - **YELLOW**：入库待查，等待家属主动确认。
4. **用户交互**：
    - **网格员**：在模块五（大屏）看到红色卡片闪烁，接收工单。
    - **家属**：通过模块四（AI 服务）获取每日生活日报或进行健康咨询。

## 5. 常见问题排查 (Troubleshooting)

- **Q: 前端大屏显示"无法连接服务器"？**
    - **A**: 请检查 `grid_monitor.html` 中的 `mainApiUrl` 是否指向了模块三运行的 IP 和端口（默认 8000）。注意跨域问题，模块三已开启 CORS，但需确保网络互通。
- **Q: 算法模块报错 **`Connection Refused`**？**
    - **A**: 检查模块三（Agent）是否已启动。模块二需要将预警结果发送给模块三。
- **Q: 数据库连接失败？**
    - **A**: 检查模块三 `database.py` 中的 MySQL 用户名密码配置，确保本地 MySQL 服务已开启。
- **Q: 如何测试报警流程？**
    - **A**: 运行模块一，选择包含"异常行为"（如全天无活动）的 CSV 数据集进行回放，观察模块三控制台日志及前端大屏变化。
